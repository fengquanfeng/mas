# Backend 项目目录结构

> masServer 后端项目 - 基于 NestJS + TypeORM + MySQL 的 RESTful API 服务

---

## 目录概览

```
backend/
├── src/                          # 源代码目录
│   ├── auth/                     # 认证模块 (JWT + Passport)
│   ├── config/                   # 配置文件
│   ├── dept/                     # 部门管理模块
│   ├── dic/                      # 字典管理模块
│   ├── log/                      # 系统日志模块
│   ├── menu/                     # 菜单管理模块
│   ├── role/                     # 角色管理模块
│   ├── user/                     # 用户管理模块
│   ├── app.controller.ts         # 根控制器
│   ├── app.module.ts             # 根模块
│   ├── app.service.ts            # 根服务
│   ├── main.ts                   # 应用入口
│   └── system.controller.ts      # 系统控制器
├── test/                         # 测试文件目录
├── dist/                         # 编译输出目录 (已排除)
├── .env                          # 环境变量配置
├── .prettierrc                   # Prettier 代码格式化配置
├── eslint.config.mjs             # ESLint 配置
├── init-database.sql             # 数据库初始化脚本
├── nest-cli.json                 # NestJS CLI 配置
├── package.json                  # npm 包配置
├── tsconfig.json                 # TypeScript 主配置
└── tsconfig.build.json           # TypeScript 构建配置
```

## 详细说明

### 1. 源代码目录 (src/)

#### 1.1 核心文件

| 文件 | 功能说明 |
|------|---------|
| `main.ts` | 应用入口文件，负责创建 NestJS 应用实例、配置中间件、启动服务器 |
| `app.module.ts` | 根模块，聚合所有功能模块，配置全局依赖注入 |
| `app.controller.ts` | 根控制器，处理根路由请求 |
| `app.service.ts` | 根服务，提供应用级别的业务逻辑 |
| `system.controller.ts` | 系统控制器，提供系统状态检查、健康监测等接口 |
| `app.controller.spec.ts` | App 控制器单元测试文件 |

#### 1.2 认证模块 (src/auth/)

| 文件 | 功能说明 |
|------|---------|
| `auth.module.ts` | 认证模块定义，导入 Passport、JWT 相关依赖 |
| `auth.controller.ts` | 认证控制器，提供登录、登出、刷新 Token 等接口 |
| `auth.service.ts` | 认证服务，处理用户验证、Token 生成与验证逻辑 |
| `dto/login.dto.ts` | 登录请求数据传输对象，定义用户名/密码校验规则 |

#### 1.3 配置模块 (src/config/)

| 文件 | 功能说明 |
|------|---------|
| `database.config.ts` | 数据库连接配置，包含 MySQL 主机、端口、用户名、密码、数据库名等 |

#### 1.4 部门模块 (src/dept/)

| 文件 | 功能说明 |
|------|---------|
| `dept.entity.ts` | 部门实体定义，包含部门 ID、名称、父部门、排序、状态等字段，支持树形结构 |

#### 1.5 字典模块 (src/dic/)

| 文件 | 功能说明 |
|------|---------|
| `dictionary.entity.ts` | 字典实体定义，包含字典编码、名称、状态等 |
| `dictionary-item.entity.ts` | 字典项实体定义，包含字典项编码、名称、值、排序、所属字典等 |

#### 1.6 日志模块 (src/log/)

| 文件 | 功能说明 |
|------|---------|
| `log.entity.ts` | 系统日志实体定义，记录用户操作、请求信息、响应结果、执行时间等 |

#### 1.7 菜单模块 (src/menu/)

| 文件 | 功能说明 |
|------|---------|
| `menu.entity.ts` | 菜单实体定义，包含菜单名称、路径、组件、图标、权限标识、父菜单、排序等，支持树形结构 |

#### 1.8 角色模块 (src/role/)

| 文件 | 功能说明 |
|------|---------|
| `role.entity.ts` | 角色实体定义，包含角色名称、编码、描述、状态、关联菜单权限等 |

#### 1.9 用户模块 (src/user/)

| 文件 | 功能说明 |
|------|---------|
| `user.entity.ts` | 用户实体定义，包含用户名、密码、昵称、头像、手机号、邮箱、状态、关联角色/部门等 |

---

### 2. 测试目录 (test/)

| 文件 | 功能说明 |
|------|---------|
| `app.e2e-spec.ts` | 端到端测试文件，测试完整请求链路 |
| `jest-e2e.json` | Jest E2E 测试配置文件 |

---

### 3. 配置文件

| 文件 | 功能说明 |
|------|---------|
| `.env` | 环境变量配置文件，存储数据库连接、JWT 密钥、端口等敏感信息 |
| `.prettierrc` | Prettier 代码格式化配置，统一代码风格 |
| `eslint.config.mjs` | ESLint 配置，代码质量检查规则 |
| `nest-cli.json` | NestJS CLI 配置文件，定义编译选项、项目元数据 |
| `tsconfig.json` | TypeScript 主配置文件，定义编译选项、路径映射 |
| `tsconfig.build.json` | TypeScript 构建专用配置，排除测试文件 |
| `package.json` | npm 包配置，定义依赖、脚本命令 |
| `init-database.sql` | 数据库初始化 SQL 脚本，创建数据库、表结构、初始数据 |
| `README.md` | 项目说明文档 |
